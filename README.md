# Qwen2.5 Inference Server (Transformers + FastAPI)

Этот проект представляет собой легкий и надежный сервер для запуска модели **Qwen2.5-32B-Instruct** на Linux с двумя GPU (например, 2x40GB). Сервер эмулирует OpenAI API, что позволяет использовать его с любыми стандартными клиентами.

## Преимущества

- **Совместимость**: Использует стандартные библиотеки `transformers` и `torch`, что гарантирует запуск даже там, где не ставится `vLLM`.
- **Multi-GPU**: Автоматическое распределение модели между картами через `accelerate`.
- **OpenAI API**: Поддержка эндпоинта `/v1/chat/completions` (включая стриминг).

## Системные требования

- **OS**: Linux
- **Python**: 3.12
- **GPU**: 2x NVIDIA GPU (минимум 40GB VRAM каждая для 32B модели в BF16)
- **CUDA**: 12.1 / 12.4

## Установка и запуск

1.  **Создайте виртуальное окружение**:
    ```bash
    python3 -m venv vllm_env
    source vllm_env/bin/activate
    ```

2.  **Установите зависимости**:
    ```bash
    pip install -r requirements.txt
    ```

3.  **Подготовьте модель**:
    Поместите веса модели в папку `qwen_model` (или измените путь в `start_server.sh`).

4.  **Запустите сервер**:
    ```bash
    chmod +x start_server.sh
    ./start_server.sh
    ```

## Тестирование

Для проверки работоспособности сервера используйте готовый скрипт:

```bash
python3 test_client.py
```

Он отправит тестовый запрос и выведет ответ модели в консоль.

## Конфигурация в `start_server.sh`

- `MODEL_PATH`: Путь к папке с моделью.
- `PORT`: Порт сервера (по умолчанию 8000).
- Модель загружается в режиме `torch.bfloat16` для оптимального баланса точности и памяти.

## Troubleshooting

- **OOM (Out Of Memory)**: Если памяти не хватает, убедитесь, что другие процессы не занимают GPU. Qwen2.5-32B в BF16 требует около 64 ГБ весов + место под активации.
- **CUDA Errors**: Убедитесь, что установленная версия `torch` соответствует вашим драйверам (рекомендуется 2.4.0+cu124).
